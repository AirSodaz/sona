# Sona 使用者指南

[English](user-guide.md) | [簡體中文](user-guide.zh-CN.md) | [繁體中文](user-guide.zh-TW.md) | [日本語](user-guide.ja.md) | [專案 README](../README.zh-TW.md)

這份指南面向桌面端使用者，幫助您完成 Sona 的安裝、首次設定、本機轉錄、文字整理與匯出。

## 1. Sona 適合做什麼

Sona 是一款以隱私優先為目標的轉錄編輯器，適合希望預設在本機完成語音轉文字流程，而不是先把音訊上傳到雲端的使用者。

如果您有以下需求，Sona 會比較適合：

- 用 `即時錄音` 記錄會議、課堂、訪談或語音筆記
- 用 `批次轉錄` 處理已有的音訊或影片檔案
- 用 `工作區`、`專案` 與 `Inbox` 組織已經儲存的錄音和匯入內容
- 在按時間軸組織的編輯器中逐段檢查文字、說話人標籤和時間戳
- 使用 `說話人檔案` 與 `說話人校對` 確認候選或匿名說話人分組
- 在自行設定服務後，使用 `LLM 潤色` 或 `翻譯`
- 當批次改寫需要回退時，從 `版本快照` 中復原選中的修改
- 將結果匯出為常見字幕或純文字格式
- 按需啟用 `語音輸入法`，把辨識結果直接輸入到其他應用

如果您主要是為了 `即時字幕` 而來，請直接檢視 `即時錄音` 章節；如果您主要想整理已經儲存的內容，可以直接跳到 `工作區、專案與 Inbox`；如果您主要需要處理說話人或版本回退，請檢視 `轉錄編輯與播放`；如果您主要想用 `語音輸入法`，請直接檢視 `匯出與設定` 中關於 `設定 > 語音輸入法` 的說明。

## 2. 安裝與啟動

對大多數使用者來說，最簡單的方式是直接安裝最新發布版本：

- 從 [GitHub Releases](https://github.com/AirSodaz/sona/releases/latest) 下載 Sona。
- 啟動應用。
- 如果您需要從原始碼建構，請參考[專案 README](../README.zh-TW.md)。

Sona 首次開啟時，可能會直接進入 `首次執行設定`。如果您暫時跳過，後續應用會繼續透過提醒橫幅提示您完成離線初始化。

### 需要 CLI？

如果您需要命令列批次轉錄，請檢視專門的 [CLI 指南](cli.zh-TW.md)。本指南只涵蓋桌面端日常使用流程。

## 3. 首次執行設定

在本機離線轉錄可用之前，Sona 需要先完成離線模型初始化。

### 前置條件

- Sona 可以正常啟動。
- 如果您準備直接下載推薦模型包，需要聯網。

### 操作步驟

1. 啟動 Sona，等待 `首次執行設定` 出現。
2. 閱讀歡迎步驟。Sona 推薦的首條成功路徑是 `麥克風 -> 即時錄音`。
3. 點擊 `繼續`。
4. 在模型步驟點擊 `下載推薦模型`。如果所需模型已經存在，介面會直接顯示 `繼續`。
5. 等待推薦離線模型下載並解壓縮完成。
6. 進入麥克風步驟並允許麥克風存取權限。
7. 如果權限被拒絕，請在處理系統權限提示後點擊 `重新請求權限`。
8. 選擇您希望 `即時錄音` 預設使用的麥克風。
9. 點擊 `前往即時錄音`。

### 完成後會發生什麼

- Sona 會應用推薦的本機離線轉錄設定。
- 應用會切換到 `即時錄音`。
- 如果設定仍未完成，提醒橫幅可以再次開啟引導。

### 說明

- 首次引導的目標是儘快讓本機離線轉錄進入可用狀態。
- 您可以在引導過程中點擊 `稍後`，之後再透過提醒橫幅返回。
- 如果您主動關閉提醒橫幅，它將不再在首頁顯示，直到設定完成。
- 後續可以在 `設定 > 模型設定` 中改用其他模型。
- 預設麥克風可以在 `設定 > 輸入裝置` 中重新調整。

## 4. 即時錄音

如果您希望邊錄音邊產生轉錄內容，請使用 `即時錄音`。

### 前置條件

- 您已經完成 `首次執行設定`，或已在 `設定 > 模型設定` 中設定 `即時錄音模型`。
- 如果要從麥克風錄音，系統已經授予麥克風權限。

### 操作步驟

1. 點擊 `即時錄音` 頁籤。
2. 在開始錄音前，從下拉框選擇輸入源：
   `麥克風` 或 `桌面音訊`。
3. 點擊 `開始錄音`。
4. 透過波形和計時器觀察採集狀態。
5. 需要臨時中斷時使用 `暫停`，要結束本次錄音時使用 `停止`。
6. 點擊 `參數設定`，按需調整 `字幕模式` 或 `語言`。
7. 如果需要懸浮字幕視窗，請開啟 `即時字幕`。
8. 如需調整字幕浮窗的置頂、點擊穿透、尺寸、寬度、顏色或啟動行為，請前往 `設定 > 字幕設定`。

### `即時字幕` 是做什麼的

- `即時字幕` 是 `即時錄音` 頁面裡的 `懸浮字幕（系統音訊）` 開關，適合您主要想把系統音訊以懸浮字幕方式顯示出來的場景。
- 它可以單獨開啟，不需要先開始錄音；如果您之後再開始 `即時錄音`，兩者也可以並行使用。
- `設定 > 字幕設定` 負責控制字幕視窗的啟動行為、置頂、點擊穿透、字型大小、寬度、顏色和背景透明度。

### 完成後會發生什麼

- 新的轉錄分段會即時出現在右側編輯器中。
- 目前活躍分段會跟隨錄音狀態變化。
- 錄音仍在進行時，Sona 也可以先把這次會話顯示成 `工作區` 裡的一個 `草稿` 條目。
- 停止錄音後，目前轉錄內容仍可繼續編輯、潤色、翻譯、匯出，並在之後從 `工作區` 重新開啟；Sona 會直接完成這條草稿，而不是再額外建立第二條記錄。
- 如果您當時不在某個專案上下文裡，儲存後的條目通常會先進入 `Inbox`。

### 說明

- 預設情況下，`Ctrl + Space` 用於開始或停止即時錄音。
- 錄音過程中按 `Space` 可以暫停或繼續。
- `參數設定` 主要控制 `字幕模式` 和 `語言`，不是完整的 LLM 潤色入口。
- 如果提示缺少模型，請重新完成引導，或到 `設定 > 模型設定` 手動設定。

## 5. 批次轉錄

如果您已經有音訊或影片檔案，並希望在背景佇列轉錄，請使用 `批次轉錄`。

### 前置條件

- 您已經在 `設定 > 模型設定` 中設定 `批次轉錄模型`。
- 輸入檔案屬於支援的音訊或影片格式。

### 操作步驟

1. 點擊 `批次轉錄` 頁籤。
2. 將檔案拖入匯入區域，或者點擊 `選擇檔案`。
3. 新增一個或多個檔案到佇列。
4. 觀察佇列側欄和目前處理項狀態。
5. 如需繼續追加檔案，點擊 `新增更多檔案`。
6. 點擊 `參數設定`，按需調整 `字幕模式` 或 `語言`。
7. 檔案處理完成後，在主編輯器中檢查結果。

### 完成後會發生什麼

- Sona 會按 `待處理`、`處理中`、`完成`、`失敗` 等狀態管理佇列。
- 已完成的檔案會載入主編輯器，供您繼續編輯、翻譯和匯出。

### 說明

- 如果未設定離線批次模型，Sona 會重新開啟引導，而不是直接開始轉錄。
- `設定 > 模型設定` 中的 `批次 VAD 分割`、`VAD Buffer Size` 與 `Max Concurrent Transcriptions` 會影響批次處理體驗。關閉批次 VAD 後，本機批次轉錄會整段辨識檔案。

## 6. 轉錄編輯與播放

當 Sona 產生轉錄分段之後，右側編輯器就是整理文字、說話人歸屬、播放時間點和已儲存版本的主要工作區。

### 前置條件

- 您已經透過 `即時錄音`、`批次轉錄` 或 `工作區` 載入了轉錄內容。

### 操作步驟

1. 在編輯器中檢視分段清單。
2. 點擊時間戳可以跳轉到對應播放位置。
3. 雙擊分段文字，或使用編輯按鈕進入編輯狀態。
4. 按 `Enter` 儲存目前分段。
5. 按 `Shift + Enter` 在游標位置拆分目前分段。
6. 使用合併按鈕將目前分段與下一段合併。
7. 使用刪除按鈕在確認後刪除分段。
8. 按 `Ctrl + F` 搜尋目前轉錄內容。
9. 如果目前條目帶有音訊，使用底部播放器進行播放、暫停、跳轉、倍速和音量控制。
10. 如果分段顯示說話人標籤，點擊說話人徽標即可把整個說話人分組繫結到某個 `說話人檔案`、展開更多全域檔案，或把該分組復原為匿名標籤。
11. 匯出前如果需要集中檢查說話人，開啟轉錄標題欄裡的 `說話人校對`。這裡可以按 `待審`、`候選建議`、`匿名`、`已辨識`、`已審閱` 或 `全部` 篩選，預覽每個分組的前幾個片段，跳到首段，確認目前標籤，應用候選檔案，手動繫結其他檔案，或復原匿名標籤。
12. 噹噹前轉錄是已儲存且非草稿的工作區條目時，可以從標題欄開啟 `版本快照`。您可以比較某個舊快照與目前轉錄的差異，復原選中的差異行，或整份回退到該快照。

### 完成後會發生什麼

- 您可以逐段修正文字。
- 時間軸與播放位置會保持聯動。
- 說話人修正會應用到同一說話人分組下的所有分段。
- 從版本快照復原前，Sona 會先儲存目前轉錄，所以回退操作本身也可以繼續追溯。

### 說明

- 只有在編輯某一段時，編輯工具欄才會顯示。
- 工具欄支援 `撤銷`、`重做`、加粗、斜體、下劃線和拆分分段。
- 搜尋結果可以在不同分段之間快速跳轉。
- 只要目前轉錄有分段，標題欄就會顯示 `說話人校對` 入口；如果這份轉錄沒有說話人後設資料，校對清單可能為空。
- `說話人檔案` 在 `設定 > 詞彙` 中建立；專案設定可以選擇目前專案啟用哪些檔案。檔案會幫助候選建議和自動匹配，但您仍然可以人工確認、重新繫結或復原匿名標籤。
- `版本快照` 只會顯示在已儲存、非草稿、且不是臨時 `current` 的工作區條目上。Sona 會在 `LLM 潤色`、`翻譯`、`重新轉錄` 等批次改寫前，以及執行復原前儲存快照。

## 7. LLM 潤色、翻譯與摘要

Sona 的 LLM 功能是可選能力。離線轉錄本身不依賴它，但 `LLM 潤色`、`翻譯` 與 `AI 摘要` 需要先在 `設定 > LLM 服務` 中完成設定。

### 前置條件

- 您已經有轉錄分段。
- 您已經在 `設定 > LLM 服務` 中設定了要使用的功能。

### LLM 服務設定步驟

1. 開啟 `設定 > LLM 服務`。
2. 在 `功能模型繫結` 中分別為 `潤色模型`、`翻譯模型` 和 `摘要模型` 指定模型。
3. 在 `Provider 憑證` 中展開要使用的 provider，填寫相應的連線資訊，例如 `Base URL`、`API Key`、`Endpoint`、`Deployment Name` 或 provider 特有欄位。
4. 如果所選模型支援，可以啟用 `深度思考模式` 並選擇 `思考程度`。
5. 點擊 `測試連線` 驗證設定。
6. 確認所需功能已經綁定模型後，再返回主介面使用。

### `LLM 潤色` 操作步驟

1. 確認 `設定 > LLM 服務` 中已經指定 `潤色模型`。
2. 點擊 `LLM 潤色` 按鈕。
3. 按需選擇：
   `LLM 潤色`、`重新轉錄`、`撤銷`、`重做`、`高階設定`。
4. 如果需要管理 `Auto-Polish`、`Auto-Polish Frequency`、`關鍵詞`、`場景預設`、`自定義上下文`，請開啟 `高階設定`。

### `翻譯` 操作步驟

1. 確認 `設定 > LLM 服務` 中已經指定 `翻譯模型`。
2. 點擊 `翻譯` 按鈕。
3. 選擇目標語言。
4. 點擊 `開始翻譯` 或 `重新翻譯`。
5. 使用 `顯示雙語` 或 `隱藏雙語` 控制編輯器中的雙語顯示。

### `AI 摘要` 操作步驟

1. 確認 `設定 > LLM 服務` 中已經啟用 `AI 摘要`，並指定 `摘要模型`。
2. 開啟任意一個已經有分段內容的轉錄。
3. 從轉錄介面開啟摘要編輯區。
4. 如果需要切換下一次產生所使用的摘要模板，可在 `通用`、`會議` 或 `課堂` 之間切換。
5. 點擊 `產生` 建立目前摘要；之後如需替換它，點擊 `重新產生`。
6. 如果需要微調措辭，可直接編輯摘要文字。焦點離開編輯區後，Sona 會自動儲存。
7. 如果需要把摘要用於其他地方，點擊 `複製`。

### 完成後會發生什麼

- `LLM 潤色` 會直接更新轉錄文字。
- `翻譯` 會把翻譯結果儲存到每個分段，並可顯示在原文下方。
- `AI 摘要` 會以“目前摘要記錄”的形式儲存在轉錄旁邊，不會改寫原文。
- 對已儲存且非草稿的轉錄，批次改寫會建立 `版本快照`，之後可以從編輯器裡重新檢視。

### 說明

- `潤色模型`、`翻譯模型` 與 `摘要模型` 是分開設定的。您可以讓它們共用同一個 provider，也可以分別設定。
- 翻譯可以使用 `Google Translate (Free)`、`Google Translate (API)` 這類專用翻譯 provider；`LLM 潤色` 則需要真正的 LLM provider 與模型。
- `AI 摘要` 也需要真正的 LLM provider 與模型，不支援 `Google Translate` 系列 provider。
- 您可以在 `設定 > LLM 服務` 中關閉 `AI 摘要`。關閉後摘要面板會隱藏，並且不會繼續產生新的摘要，但已有 sidecar 摘要資料會保留。
- 如果 `AI 摘要` 還沒有完成 LLM 設定，您仍然可以先開啟摘要編輯區，手動編寫或修改摘要文字。
- 每份轉錄只會保留一條“目前摘要記錄”。切換模板只會改變下一次產生所使用的模板，或專案預設模板，不會為每個模板各自儲存一份獨立摘要。
- 目前支援的翻譯目標語言包括 `簡體中文`、`English`、`Japanese`、`Korean`、`French`、`German`、`Spanish`。
- 只有噹噹前轉錄來自已儲存的工作區條目時，選單中才會出現 `重新轉錄`。
- 目前摘要結果支援直接編輯、自動儲存和複製複用。
- 如果潤色、翻譯或重新轉錄改動過多，可以從轉錄標題欄開啟 `版本快照`，復原選中的差異行，或整份回退到某個舊快照。

## 8. 匯出轉錄

只要編輯器裡至少有一個分段，就可以匯出。

### 前置條件

- 目前編輯器中已經有轉錄內容。

### 操作步驟

1. 點擊頂部欄中的 `匯出轉錄` 按鈕。
2. 在 `匯出轉錄` 彈窗中填寫 `檔名`。
3. 選擇 `匯出目錄`。
4. 選擇輸出格式：
   `SubRip (.srt)`、`WebVTT (.vtt)`、`JSON (.json)`、`Plain Text (.txt)`。
5. 選擇匯出模式：
   `原文`、`譯文`、`雙語`。
6. 點擊 `匯出`。

### 完成後會發生什麼

- Sona 會把轉錄內容寫入您選擇的路徑和格式。
- 如果已有翻譯內容，您可以匯出純譯文或雙語版本。

### 說明

- 只有在目前轉錄至少包含一個翻譯分段時，`譯文` 和 `雙語` 才可用。
- `原文` 模式始終可用。
- 如果匯出結果或交付前檢查需要說話人標籤，請先執行 `說話人校對`，把候選和匿名分組確認好。

## 9. 工作區、專案與 Inbox

如果您希望在不離開主編輯面的前提下整理已經儲存的錄音和匯入內容，就使用 `工作區`。

### 三種瀏覽範圍分別代表什麼

- `全部內容` 用來跨 `Inbox` 和各個專案總覽所有已儲存內容。
- `Inbox` 用來承接尚未分配到專案的錄音和匯入內容。
- 單一專案提供聚焦後的工作區、專案級預設值，以及針對新工作的快捷入口。

### 操作步驟

1. 開啟 `工作區`。
2. 使用左側邊欄在 `全部內容`、`Inbox` 和各個專案之間切換。
3. 如果您想為某個客戶、課程、會議系列或長期主題建立獨立空間，點擊 `新建專案`。
4. 輸入專案名稱，並按需填寫簡短描述後建立。新專案建立時不會立刻設定自定義圖示。
5. 當您希望新的錄音或匯入內容自動歸入某個專案時，先開啟那個專案。
6. 在專案頂部使用 `開始即時錄音` 或 `開啟批次匯入`，即可在保留目前專案上下文的前提下開始新工作。
7. 點擊任意已儲存條目，即可在內建詳情面板中直接開啟並繼續編輯，不必離開 `工作區`。
8. 使用搜尋、篩選、排序，以及 `清單檢視` / `網格檢視` / `表格檢視` 縮小目前瀏覽範圍。
9. 如果您要在 `Inbox` 與專案之間移動內容，或一次刪除多項內容，請先進入選擇模式。

### `專案設定`

1. 先開啟某個專案，再點擊 `專案設定`。
2. 修改專案名稱和描述。
3. 如果需要，自定義專案圖示。專案圖示是在這裡修改的，不是在 `新建專案` 彈窗裡設定。
4. 設定該專案內工作的預設值：
   `預設摘要模板`、`預設翻譯語言`、`預設潤色場景`、可選的 `預設潤色上下文`，以及 `匯出檔名字首`。
5. 如果您希望專案使用特定的辨識、清洗、潤色或說話人匹配規則，可以單獨啟用或關閉該專案的 `文字替換`、`熱詞`、`潤色關鍵詞` 與 `說話人檔案`。
6. 點擊 `儲存` 保留修改；如果點擊 `刪除專案`，專案本身會被移除，裡面的內容會回到 `Inbox`。

### 完成後會發生什麼

- `工作區` 會成為您整理已儲存內容、同時又能隨時繼續編輯的主要入口。
- 只要目前專案處於啟用狀態，專案級預設值就會在該專案內生效。
- 在專案內發起的新工作會保留該專案上下文，除非您之後手動移動。

### 說明

- `全部內容` 適合跨專案瀏覽，但它不會替代目前專案的預設值。
- 當您不在任何專案內工作時，新的錄音和匯入內容會預設先進入 `Inbox`。
- 即時錄音仍在進行時，這裡也可能先出現一個帶 `草稿` 標記的條目。停止錄音後，Sona 會直接把這條草稿完成為正式條目。
- 專案圖示支援系統圖示、推薦 emoji，以及自定義 emoji。
- 已儲存的錄音和匯入條目也支援在各自的重新命名流程中修改標題和圖示，但那與 `專案設定` 是兩套入口。
- 如果您在某個專案的轉錄中手動繫結了全域說話人檔案，Sona 可以把該檔案加入目前專案啟用的說話人檔案中，方便後續同專案內容繼續作為候選使用。
- 如果您在專案設定抽屜裡有未儲存更改，切換到別的瀏覽範圍前，Sona 會先詢問您是否放棄這些修改。

## 10. 匯出與設定

如果您準備把檔案匯出，就使用 `匯出`；如果您要管理應用級預設行為，就使用 `設定`。如果您需要重新開啟或整理已儲存內容，請回到 `工作區`，而不是再尋找另一套獨立入口。

### 匯出提醒

- 當轉錄整理完成後，使用頂部欄中的 `匯出` 按鈕。
- 上文 `8. 匯出轉錄` 已經列出目前支援的格式與模式。

### 建議優先了解的設定區域

- `設定 > 儀表板`
  全域內容概覽、說話人覆蓋率與 LLM 使用趨勢
- `設定 > 常規`
  主題、介面語言、字型、托盤行為、更新檢查，以及 `診斷` 與 `備份與復原`
- `設定 > 輸入裝置`
  麥克風選擇、系統音訊裝置選擇、麥克風增強、錄音時靜音
- `設定 > 字幕設定`
  即時字幕啟動行為、點擊穿透鎖定、置頂、字型大小、寬度、顏色、背景透明度
- `設定 > 語音輸入法`
  啟用 `語音輸入法`、設定全域快捷鍵、選擇 `按住說話` 或 `切換模式`，並檢視就緒狀態
- `設定 > 模型設定`
  `即時錄音模型`、`批次轉錄模型`、`轉錄設定`、`ITN`、`批次 VAD 分割`、`VAD Buffer Size`、`Max Concurrent Transcriptions`、`復原預設設定`，以及可下載的辨識模型、標點模型、說話人模型、VAD 模型
- `設定 > 詞彙`
  `文字替換`、`熱詞`、潤色關鍵詞規則集、潤色上下文預設、摘要模板與 `說話人檔案`
- `設定 > 自動化`
  監控資料夾規則，可在 Sona 執行時自動轉錄、潤色、翻譯並匯出新媒體檔案
- `設定 > LLM 服務`
  功能模型繫結、深度思考選項與 Provider 憑證
- `設定 > 快捷鍵`
  即時錄音、播放、搜尋、工作區導航與編輯器快捷鍵
- `設定 > 關於`
  原始碼、日誌與更新相關入口

### `語音輸入法`

- `語音輸入法` 適合在聊天視窗、文件編輯器、表單等其他應用中直接輸入文字。
- 前往 `設定 > 語音輸入法`，啟用 `語音輸入法`，設定全域快捷鍵，並選擇 `按住說話` 或 `切換模式`。
- `按住說話` 更適合短句輸入；`切換模式` 更適合連續口述，按一次開始，再按一次停止。
- `語音輸入法` 依賴同一套離線即時轉錄能力，因此也需要先設定可用的 `即時錄音模型`、所需的 `VAD` 模型，以及可用的輸入裝置。
- 同一頁還會提示目前的阻塞點究竟來自快捷鍵、模型、VAD、輸入裝置，還是背景預熱狀態。

### 診斷與備份

- 在 `設定 > 常規` 中，可以透過 `診斷` 檢查本機轉錄鏈路、執行時就緒狀態和打包環境。
- 同一頁中的 `備份與復原` 可以匯出或匯入輕量歸檔，其中包含設定、工作區、輕量歷史轉錄與摘要、自動化狀態，以及儀表板 LLM 使用記錄。
- 輕量備份會復原文字歷史和摘要，但不會包含原始音訊檔案；復原後的條目可能可以重新閱讀和編輯，但無法播放音訊。
- `WebDAV 雲同步` 位於 `備份與復原` 內部。它會把憑證儲存在目前裝置上，並幫助您手動上傳或復原備份歸檔。

### 通知中心

- 頂部通知中心會把更新、`復原中心` 與自動化結果集中到一個入口。
- 當 Sona 偵測到中斷的批次處理/自動化任務、出現可重啟的更新，或自動化規則成功/失敗時，都可以從這裡直接處理。
- `復原中心` 用於繼續或丟棄中斷的批次處理與自動化條目；`診斷` 與 `備份與復原` 仍然位於 `設定 > 常規`。

### 說明

- 在 `設定 > 詞彙` 中，`熱詞` 按每行一個的方式填寫，也支援 `關鍵詞 :2.0` 這類帶權重的寫法；目前這項能力對 Transducer 與 Qwen3 ASR 模型尤其相關。
- 在 `設定 > 詞彙` 中，`文字替換` 適合用來統一術語或修正重複出現的辨識結果。
- 在 `設定 > 詞彙` 中，`說話人檔案` 會儲存已知說話人的本機參考樣本。檔案會根據可用樣本數量和時長顯示為可自動匹配、可作為候選建議或尚未就緒；這些狀態影響匹設定信度，不影響您手動繫結檔案。

## 11. 常見問題與排障

### Sona 一直提示我完成首次設定

- 開啟提醒橫幅，把模型和麥克風步驟完成。
- 如果您之前跳過了引導，請確認 `設定 > 模型設定` 中已經設定好 `即時錄音模型` 和 `批次轉錄模型`。
- 如果您曾主動關閉提醒橫幅，請在 Sona 再次提示時從相關入口重新進入引導。

### `即時錄音` 無法開始

- 檢查系統麥克風權限。
- 確認已經設定 `即時錄音模型`。
- 確認目前輸入源是否正確：`麥克風` 或 `桌面音訊`。

### `批次轉錄` 無法開始

- 確認已經設定 `批次轉錄模型`。
- 確認副檔名屬於支援格式。
- 如果 Sona 提示格式不支援，請先轉換檔案格式再重試。

### `LLM 潤色` 或 `翻譯` 不可用，或者執行失敗

- 確認 `設定 > LLM 服務` 中已經填寫正確的 provider 憑證。
- 確認功能本身已經綁定模型：潤色需要 `潤色模型`，翻譯需要 `翻譯模型`。
- 先使用 `測試連線` 驗證連通性。
- 如果您使用自定義介面地址或本機服務，例如 `Ollama`，請先確認那個服務本身可存取。

### 找不到 `Auto-Polish`

- 請開啟 `LLM 潤色 > 高階設定`，在那裡可以設定 `Auto-Polish`、頻率、關鍵詞、場景預設和自定義上下文。

### 匯出時只有 `原文`

- 只有噹噹前轉錄已經包含翻譯文字時，`譯文` 和 `雙語` 才會出現。

### `說話人校對` 是空的，或者沒有候選人

- `說話人校對` 依賴目前轉錄已有的說話人後設資料。如果這份轉錄沒有說話人歸屬資訊，清單可能為空。
- 候選建議取決於 `設定 > 詞彙` 中的 `說話人檔案`、已匯入的參考樣本，以及目前專案啟用了哪些檔案。
- 只要編輯器中已經顯示說話人標籤，您仍然可以點擊分段上的說話人徽標手動繫結檔案。

### 找不到 `版本快照`

- `版本快照` 只會出現在已經儲存到工作區、且包含轉錄分段的條目上。
- 臨時 `current` 轉錄和仍在進行中的即時錄音草稿不會顯示這個入口。
- Sona 會在 `LLM 潤色`、`翻譯`、`重新轉錄` 等批次改寫前，以及從其他快照復原前建立快照。

### 復原備份後能看到文字，但沒有音訊播放

- 備份歸檔刻意保持輕量，只包含設定、工作區、輕量歷史轉錄與摘要、自動化狀態，以及儀表板 LLM 使用記錄。
- 原始音訊檔案不會進入備份，因此復原後的條目可能可以閱讀和編輯，但除非音訊透過其他路徑可用，否則不會顯示播放能力。

### 為什麼新內容會先進入 `Inbox`

- `Inbox` 是錄音和匯入內容的預設承接位置，專門用來存放尚未分配到專案的內容。
- 如果您希望新內容自動留在某個專案裡，請先開啟那個專案，再開始 `即時錄音` 或 `批次轉錄`。
- 如果內容已經進入 `Inbox`，您之後仍然可以在 `工作區` 中把它移動到目標專案。

### 刪除專案後，裡面的內容會怎樣

- 刪除專案只會移除專案本身，不會刪除專案裡的錄音或匯入內容。
- Sona 會把這些內容移回 `Inbox`，您之後仍然可以繼續使用或重新分配它們。

### 為什麼我還在錄音時就看到了一個 `草稿` 條目

- 即時錄音進行中時，Sona 可能會先建立一個可見的 `草稿` 條目，讓這次會話提前在 `工作區` 裡佔好位置。
- 當您停止錄音後，Sona 會直接完成這條草稿，而不是再額外建立第二條記錄。
- 在採集中，請把這條草稿理解為目前的即時會話；錄音結束後，正式轉錄仍然會從同一入口繼續使用。

### `即時字幕` 沒有顯示出來

- 請先回到 `即時錄音` 頁面，確認已經開啟 `即時字幕`；`設定 > 字幕設定` 只負責視窗行為和外觀，不負責啟動入口。
- 如果您只是想顯示系統音訊字幕，不必先開始錄音，直接開啟 `即時字幕` 即可。
- `即時字幕` 依賴同一套離線即時轉錄能力，因此也需要先設定可用的 `即時錄音模型`。

### `語音輸入法` 無法工作

- 先在 `設定 > 語音輸入法` 中啟用 `語音輸入法`。
- 確認頁面上的快捷鍵、模型、VAD、輸入裝置與就緒狀態都符合預期。
- 語音輸入法依賴同一套離線即時轉錄能力，因此也需要先設定可用的即時轉錄模型。

### 去哪裡復原中斷的批次處理或自動化任務

- 如果 Sona 提示有待復原專案，請先開啟頂部通知中心。
- 使用 `復原中心` 繼續或丟棄中斷的批次處理與自動化任務。
- 如果您實際要找的是診斷或備份入口，請前往 `設定 > 常規`。

### 看不到播放控制元件

- 只有噹噹前轉錄關聯了音訊來源時，底部播放器才會顯示，例如已儲存的錄音條目或已處理完成的檔案。

### 我想從原始碼建構或參與開發

- 請檢視[專案 README](../README.zh-TW.md)中的原始碼建構與開發命令說明。
