# Nightly Release Workflow

The Nightly Release workflow automatically builds, signs, packages, and publishes pre-release builds of Sona every day. It also generates and updates the update metadata (`updater.json`) for the Tauri auto-updater.

Workflow File: [.github/workflows/nightly.yml](file:///c:/Users/asoda/projects/sona/.github/workflows/nightly.yml)

---

## Triggers

1. **Schedule (Cron)**: Runs automatically every day at `18:00 UTC` (2:00 AM Beijing Time / 10:00 AM PST).
2. **Manual Dispatch (`workflow_dispatch`)**: Can be manually triggered from the GitHub Actions tab.
   - **Inputs**:
     - `force` (boolean, default: `false`): Force a build and release even if there are no new commits since the last nightly release.

---

## Concurrency

The workflow runs under the concurrency group `nightly-release` with `cancel-in-progress: true`. If a new run is triggered while a previous run is still active, the previous run is cancelled to save resources.

## Workflow Jobs

### 1. `check-commits`
* **Runs on**: `ubuntu-latest`
* **Purpose**: Avoid redundant builds if the codebase hasn't changed.
* **Flow**:
  1. Checks the current commit SHA (`github.sha`).
  2. Resolves the commit SHA of the existing `nightly` tag using `git ls-remote`.
  3. If the current SHA is the same as the nightly tag SHA (and `force` is not set to `true`), the workflow exits early with `should_build=false`.

### 2. `prepare`
* **Runs on**: `ubuntu-latest`
* **Purpose**: Compute nightly version identifiers.
* **Flow**:
  1. Reads the base version from `platforms/desktop/frontend/package.json`.
  2. Generates the nightly version string: `${base_version}-${github.run_number}`.
  3. Computes the build date in `YYYY-MM-DD` format (UTC).

### 3. `build-tauri`
* **Runs on**: Matrix platforms (`macos-latest`, `ubuntu-22.04`, `windows-latest`)
* **Purpose**: Compiles the Sona application, signs the bundle, and generates signatures.
* **Matrix Targets**:
  - **macOS**: `macos-latest` (builds `aarch64-apple-darwin`, `x86_64-apple-darwin`, and `universal-apple-darwin` targets).
  - **Linux**: `ubuntu-22.04` (default target, compiles `.deb`, `.rpm`, `.AppImage`).
  - **Windows**: `windows-latest` (default x64 target and `aarch64-pc-windows-msvc` ARM64 target).
* **Key Steps**:
  1. **Dependencies**: Installs native dependencies (e.g., GTK/Webkit2gtk on Linux).
  2. **Rust & Node Setup**: Configures Rust toolchain, enables `corepack`, and installs dependencies via `pnpm`.
  3. **Channel Patching**: Invokes `node platforms/desktop/scripts/patch-channel.js --channel nightly --version <version>` to update Tauri configuration with the nightly channel and version.
  4. **Sherpa-Onnx Libs**: Downloads and extracts the correct prebuilt `sherpa-onnx` shared libraries for each target platform.
  5. **Tauri Build**: Builds Sona via `node platforms/desktop/scripts/tauri.js build`. Signs binaries using the provided signing key.
  6. **Asset Renaming**: Renames macOS build artifacts to append the architecture (`aarch64`, `x64`, or `universal`) to prevent conflicts in the release upload.
  7. **Artifact Upload**: Uploads generated binaries and update signatures (`.sig` files) as workflow run artifacts.

### 4. `publish-nightly`
* **Runs on**: `ubuntu-latest`
* **Purpose**: Update the git tag, generate update metadata, and publish the release.
* **Key Steps**:
  1. **Download Artifacts**: Pulls down all files built by `build-tauri`.
  2. **Changelog Generation**: Generates a git commit log between the previous nightly tag SHA and the current commit.
  3. **Move Tag**: Updates the `nightly` Git tag via the GitHub API to point to the current commit.
  4. **Build `updater.json`**: Reads the updater signatures (`.sig` files) and constructs a Tauri-compatible `updater.json` containing the URLs to the newly uploaded nightly assets and their signatures.
  5. **Create/Update Release**: Uses `ncipollo/release-action` to update the existing pre-release named `nightly` with the new binaries and the generated `updater.json`.

## Required Secrets

The following credentials must be set in GitHub repository secrets for the workflow to complete successfully:

1. **`TAURI_SIGNING_PRIVATE_KEY`**: The private key generated by Tauri CLI, used to sign the application bundles.
2. **`TAURI_SIGNING_PRIVATE_KEY_PASSWORD`**: The passphrase protecting the Tauri signing private key.
3. **`github.token` / `GH_TOKEN`**: The built-in GitHub Actions token, used to perform API calls (moving tags, reading files) and uploading release assets.
